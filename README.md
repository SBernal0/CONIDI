# Plataforma CONIDI — Control Niño Sano Digital

Resumen
-------
Este repositorio contiene todo el material relacionado con el proyecto de título "Plataforma de Inteligencia de Negocios para la Gestión del Programa de Niño Sano en CESFAM (CONIDI) – Control Niño Sano Digital". El objetivo del proyecto es diseñar e implementar una plataforma que facilite la gestión, análisis y visualización de datos clínicos y de seguimiento del programa "Niño Sano", apoyando la toma de decisiones en los centros de salud familiar (CESFAM).

Estado del proyecto
-------------------
- Estado actual: Desarrollo (especificar: prototipo / implementación / pruebas / desplegado)

Objetivos
---------
- Objetivo general: Implementar una plataforma que integre captura de datos, procesamiento y visualización para apoyar la gestión del programa "Niño Sano".
- Objetivos específicos:
  - Desarrollar la interfaz de usuario para el registro y seguimiento de controles.
  - Implementar la API y la lógica de negocio para el manejo de datos clínicos.
  - Construir paneles de inteligencia de negocios (reportes y visualizaciones).
  - Validar la solución mediante pruebas funcionales y con usuarios representativos.

Alcance
-------
La plataforma contempla:
- Registro y actualización de controles clínicos infantiles.
- Gestión de usuarios y roles (profesionales de salud, administrativo).
- Visualizaciones y reportes para seguimiento poblacional y por paciente.
- Exportación básica de datos para análisis complementarios.

Estructura del repositorio
--------------------------
- README.md — Documento principal (este archivo).
- frontend/ — Código front-end (HTML, CSS, assets).
- backend/ — Código back-end (API en Python, scripts).
- docs/ — Documentación adicional (diagramas, especificaciones).
- data/ — Datos de ejemplo o conjuntos de prueba (si aplica).
- requirements.txt — Dependencias Python (en la raíz o en backend/).
- .env.example — Ejemplo de variables de entorno (si aplica).

Instalación y ejecución (resumen)
---------------------------------
Requisitos previos:
- Python 3.8+ (para backend)
- Navegador moderno (para frontend)
- Otras herramientas: virtualenv, pip, node/npm

Pasos generales (detallados en backend/README.md y frontend/README.md):
1. Clonar el repositorio:
   git clone https://github.com/SBernal0/CONIDI.git
2. Backend: crear entorno virtual, instalar dependencias y configurar variables de entorno.

Uso
---
Se recomienda consultar los README específicos de backend y frontend para instrucciones de uso detalladas, endpoints disponibles y ejemplos de carga de datos.
